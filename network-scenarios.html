Network Security Scenarios - Incident Reporting

Overview

This page showcases hands-on scenario-based activities from the Connect and Protect: Networks and Network Security course. Each scenario represents a simulated network security incident, such as a Denial-of-Service (DoS) attack, DNS server outage, and other potential threats.

The purpose of these exercises is to practice identifying vulnerabilities, analyzing network impacts, and documenting actionable cybersecurity incident reports. These scenarios provide insight into real-world incident response and help build practical skills in network defense and security analysis.

Scope

  • Review of security controls and policies
  • Risk assessment based on the CIA Triad
  • Identification of control gaps
  • Internal audit methodology

Frameworks & Concepts Applied

  • NIST Cybersecurity Framework (CSF)
  • NIST SP 800-53 (Security Controls)
  • CIA Triad (Confidentiality, Integrity, Availability)
  • OWASP Secure Design Principles

Key Activities

  • Evaluated existing security controls
  • Identified risks and control deficiencies
  • Assessed impact on organizational assets
  • Documented findings and recommendations
