Threat Modeling with PASTA for Secure Application Design

Overview:

This project applies the PASTA (Process of Attack Simulation and Threat Analysis) framework to evaluate the security risks of a new shopping application. The analysis focuses on identifying potential threats, vulnerabilities, and attack paths before the application is launched.

Scope:

This case study focuses on a fictional sneaker marketplace application designed to connect buyers and sellers. The scope includes key system components such as user authentication, API communication, payment processing, data storage, and encryption mechanisms. The analysis is centered on identifying security risks across these technologies and user interactions.

Methodology:

This project follows the PASTA threat modeling framework, a structured approach used to identify and analyze security risks in applications before deployment. The methodology is divided into seven stages, starting with understanding business objectives and system functionality, followed by identifying the technical scope and data flows. It then focuses on threat identification, vulnerability analysis, and attack modeling using supporting diagrams. Finally, the process concludes with recommending security controls to mitigate identified risks and reduce the likelihood of security incidents.

Scenario:

This scenario is based on a fictional company:

You’re part of the growing security team at a company for sneaker enthusiasts and collectors. The business is preparing to launch a mobile app that makes it easy for their customers to buy and sell shoes.

You are performing a threat model of the application using the PASTA framework. You will go through each of the seven stages of the framework to identify security requirements for the new sneaker company app.

Analysis / Findings:

Stage I: Define Business and Security Objectives
  • Enable a secure and seamless platform for users to buy and sell sneakers, including account registration, authentication, and transaction processing.
  • Protect sensitive user data, including personal information and payment details, to maintain trust and comply with legal and regulatory requirements.
  • Ensure a reliable and user-friendly experience that supports communication between buyers and sellers while maintaining system integrity and availability

Stage II: Define the Technical Scope
List of technologies used by the application:
  • API
  • PKI
  • AES
  • SHA-256
  • SQL

APIs and SQL databases are prioritized because they handle data exchange and storage, making them primary targets for attacks such as injection and unauthorized access. Securing these components is critical to protecting sensitive user and transaction data, while encryption and hashing mechanisms support overall data protection.


Stage III: Decompose Application

The data flow diagram illustrates how customers interact with the sneaker marketplace application through two primary processes: searching for sneakers and managing user accounts. The “Search Sneakers” process accesses data from the sneaker inventory database, while the “Manage Account” process reads and updates information stored in the user profiles database.

Analyzing the data flow helped identify areas where sensitive information could be exposed during processing and storage. Particular attention was given to protecting user account data, securing database interactions, and ensuring that data exchanged between users and the application is handled safely.

Figure 1: Data flow diagram showing customer interactions, application processes, and database communication within the sneaker marketplace application.


Stage IV: Threat Analysis

SQL Injection Vulnerability: The application may be vulnerable to SQL injection attacks if user inputs, such as search queries or login fields, are not properly validated or parameterized. Attackers could manipulate database queries to access, modify, or delete sensitive information stored in the system.

Broken Authentication & Authorization in API: Weak authentication mechanisms or improper authorization controls within APIs could allow attackers to gain unauthorized access to user accounts, view sensitive data, or perform unauthorized actions such as modifying listings and transactions.


Stage V: Vulnerability Analysis

Improper Input Validation in Database Queries: The application may fail to properly validate or sanitize user inputs before processing SQL queries. The use of dynamic queries instead of parameterized statements increases the risk of SQL injection vulnerabilities.

Weak Authentication and Authorization Controls in API: The API may lack strong authentication protocols and proper access control validation, potentially allowing unauthorized users to access or manipulate protected resources and user data.


Stage VI: Attack Modeling

The attack tree outlines several ways a threat actor could compromise the sneaker marketplace application. The analysis focused on SQL injection, unauthorized account access through weak API controls, and service disruption caused by abuse of application resources.

Figure 2: Attack tree diagram illustrating potential attack paths that could be used to compromise the sneaker marketplace application, including SQL injection, unauthorized account access, and service disruption.


Stage VII: Risk Analysis and Security Controls

Use Parameterized Queries: Implement parameterized queries to prevent SQL injection attacks by ensuring user input is treated strictly as data, reducing the risk of unauthorized database access.

Enforce Least Privilege Access: Limit database and system permissions to only what is necessary, reducing the impact of a potential compromise and preventing unauthorized data access or modification.

Strong Authentication and Authorization: Implement secure authentication mechanisms and enforce proper access controls to ensure only authorized users can access system resources and perform sensitive actions.

Multi-Factor Authentication (MFA) and Strong Password Policies: Require strong passwords and MFA to reduce the likelihood of unauthorized access due to compromised credentials or bruteforce attacks.

Key Takeaways

This project demonstrated how the PASTA threat modeling framework can be used to identify security risks, analyze vulnerabilities, and recommend mitigation strategies before an application is deployed. By evaluating potential attack paths and implementing security controls, the overall security posture of the sneaker marketplace application can be significantly improved.
